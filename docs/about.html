Specifications & Limitations

Bilingual Support

CirPath ships with complete English and Japanese translations for every visible string: window titles, status messages, error prompts, menus, key-binding references, and the editor's shortcut grid. You can switch display languages instantly from Menu -> General.

High-Contrast Color Palette

The UI intentionally avoids blue/cyan tones, which are often hard to read on dark terminal background schemes. The palette is carefully restricted to:

  • White background + black text (normal status bars, prompts, title bar)
  • Red background + white text (error messages, delete-confirmations)
  • Yellow (bold) (directories in the File Browser)
  • Dark gray (line numbers, unfocused menu panels)

Error Display Behavior

Errors (such as failed saves or unsuccessful searches) are clearly flagged in the status bar with an unmistakable red background color.

[ERROR] File could not be saved.

Any subsequent keystroke automatically clears the message instantly, ensuring stale errors never linger on your screen and disrupt your workspace.

Known Limitations

To keep the application highly performant and codebase structured, the following advanced editing features are currently not implemented:

  • Multiple buffers / Tabbed editing
  • Syntax highlighting
  • Undo / Redo operations
  • Text selection (marking) for copy - only whole-line cut/paste (Ctrl+K / Ctrl+U) is supported

These features are straightforward to add on top of the existing editor.rs structure if needed as the project scales.
