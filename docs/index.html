CirPath

CirPath は、nano や pico にインスパイアされた、Rust製のTUI（ターミナルユーザーインターフェース）テキストエディタです。

エディタ、ファイルブラウザ、メニューの3つのウィンドウを1つのアプリに統合しており、Ctrl+E / Ctrl+Q でシームレスに切り替えることができます。枠線で囲まれた境界線はなく、nano のようにシンプルなタイトルバーと反転表示のステータスバーのみで無駄のないインターフェースが構築されています。

画面上のすべてのテキストはプレーンなASCIIまたは標準的な日本語（JIS範囲）文字を使用しており、絵文字や特殊な外字など環境に依存する文字を排除しています。表示言語は英語と日本語に完全対応し、メニュー画面からリアルタイムに切り替え可能です。

🚀 ビルドと実行

cargo build --release
./target/release/cirpath

🛠️ ツールチェーンに関する注意事項

本プロジェクトは、最小要件として rustc 1.75 / cargo 1.75 を対象としています。互換性を保つため、Cargo.toml にて依存関係のバージョンを厳密に固定しています（ratatui = "=0.26.3", crossterm = "=0.27.0" など）。より新しいRustツールチェーンを使用している場合は、cargo update を実行してこれらの固定を安全に解除できます。

🎨 カラーパレット

UIは、黒背景のターミナルで視認性が低下しやすい青やシアン系の色を意図的に避けてデザインされています。

白 / 黒
通常ステータスバー、プロンプト、タイトルバー
赤 / 白
エラーメッセージ、削除確認プロンプト
黄色 (太字)
ファイルブラウザ内のディレクトリエントリ
ダークグレー

⚠️ 現在の制限事項

以下の機能は現時点では未実装ですが、既存の editor.rs の構造の上にスムーズに追加できるように設計されています。

  • 複数バッファ / タブ表示
  • シンタックスハイライト
  • 元に戻す (Undo) / やり直し (Redo)
  • コピー用のテキスト選択（マーク） - 現時点では行全体の切り取り/貼り付け（Ctrl+K / Ctrl+U）のみ対応

エディタ画面

エディタの下部には、従来の nano や UW PICO のスタイルを踏襲した「2行×6列のショートカットグリッド」が常時表示されます。これにより、利用可能なコマンドが常に一目で確認できます。

🌐 グローバル操作（全画面共通）

キー アクション
Ctrl+E 次のウィンドウへ切り替え（エディタ → ブラウザ → メニュー → エディタ... のサイクル）
Ctrl+Q 前のウィンドウへ切り替え
Ctrl+G ヘルプを表示（メニュー内の「キーバインド」カテゴリへダイレクトにジャンプ）

📝 エディタ操作（nano互換機能）

キー アクション
表示可能文字 カーソル位置に文字を挿入（日本語等のIME入力の確定処理はOS側で完全に処理されます）
Enter 改行を挿入
Backspace / Delete カーソルの前、またはカーソル位置の文字を削除（行の境界を越えた結合に対応）
矢印キー / Home / End カーソルを移動
Page Up / Page Down ページ単位でスクロール移動
Ctrl+S ファイルを上書き保存
Ctrl+O 名前を付けて保存 - ファイル名入力プロンプトを表示 (Write Out)
Ctrl+R カーソル位置に別ファイルの内容を挿入 (Read File)
Ctrl+W 検索 (Where Is)。空のプロンプトでEnterを押すと前回の検索をリピートします
Ctrl+\ 置換 - 検索文字列と置換文字列を入力し、すべての一致箇所を置換
Ctrl+_ 指定した行番号へジャンプ
Ctrl+K 現在の行を切り取り（連続して押すと1つのクリップボードバッファに蓄積）
Ctrl+U 最後に切り取ったテキストをカーソル位置に貼り付け (UnCut)
Ctrl+C 現在のカーソル位置（行/列）と合計行数をステータスバーに表示
Ctrl+X 終了 - 未保存の変更がある場合は確認プロンプトを表示
Esc 実行中のプロンプトをキャンセル（検索・置換・保存・行移動・ファイル挿入時）

ファイルブラウザ画面

内蔵のファイルブラウザを使用することで、ディレクトリの移動やファイル・フォルダの作成・削除をエディタを閉じることなくスムーズに行えます。

項目数が膨大なディレクトリでも、選択項目が常に画面内に収まるようリストが自動的にスクロールします。

📁 ファイルブラウザ操作

キー アクション
Up / Down 選択項目の移動
Enter / Right 選択したディレクトリに入る、または選択したファイルをエディタで開く
Backspace / Left / u 1つ上の親ディレクトリへ移動
g 絶対パスまたは相対パスを手動入力して直接ジャンプ（起動時のディレクトリ外も移動可能）
n 現在のディレクトリに新しいファイルを作成
N (Shift+n) 現在のディレクトリに新しいフォルダを作成
r 選択中のファイルまたはフォルダの名前を変更
d 選択中のファイルまたはフォルダを削除（実行前に y/n の確認プロンプトを表示）
Ctrl+H 隠しファイル（ドットファイル）の表示 / 非表示を切り替え
Esc 実行中のプロンプトをキャンセル（パス入力・新規作成・名前変更時）

💡 便利なデフォルト機能

メニュー設定の「ファイル選択時にエディタへ自動切り替え (Auto-switch to editor when a file is selected)」が有効な場合（デフォルト）、ブラウザでファイルを選択してEnterを押すと、自動的にフォーカスがエディタ画面へと移行します。

メニューと設定画面

Wordスタイルにインスパイアされた「2ペイン構成」の設定画面を提供します。左側にカテゴリ、右側に詳細設定項目が並ぶ直感的なレイアウトです。

メニューから変更した設定は即座にアプリケーションに反映され、自動的に ~/.config/cirpath/config.toml へ保存・永続化されます。

⚙️ メニュー画面の操作

キー アクション
Left / Right 左ペイン（カテゴリ選択）と右ペイン（設定詳細）の間でフォーカスを切り替え
Up / Down 現在フォーカスされているペイン内での選択項目の移動
Enter 左ペインからは右ペインへ移動。右ペインからは選択した設定の適用・トグル切り替え

📂 提供されているカテゴリ

  • General (一般): 表示言語の切り替えなどを行う基本設定
  • Editor (エディタ): エディタの挙動や表示に関する設定
  • File Browser (ファイルブラウザ): ブラウザの動作やフィルタに関する設定
  • Key Bindings (キーバインド): すべてのショートカットキーを確認できる完全なリファレンス
  • Config File (設定ファイル): config.toml をエディタで直接開き、手動で細かく編集可能
  • About (概要): アプリケーションのバージョン情報やクレジット

🌍 完全な多言語サポート

CirPathは英語と日本語の完全なローカライズ文字列を内蔵しています。ウィンドウタイトル、ステータス、エラー、プロンプトはもちろん、メニューの項目やエディタ最下部のショートカットグリッドに至るまで瞬時に切り替わります。設定は Menu -> General -> "Switch display language" から即座に変更可能です。

🛠️ 開発者向け：機能の拡張方法

新しい機能を追加する場合は、該当する画面のモジュール（editor.rs / browser.rs / menu.rs）を拡張し、新しい設定項目を Config 構造体へ定義した上で、menu.rs 内の items_for および apply_selection に処理を組み込みます。各画面モジュールは疎結合で自己完結しているため、通常は他のモジュールに影響を与えることなく機能拡張が行えます。
