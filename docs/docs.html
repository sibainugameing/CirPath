操作ドキュメント

CirPathはキーボードショートカットを巧みに使うことで、思考を妨げずにすべての操作を行えます。

グローバルコントロール

どの画面にいても機能する、CirPathの核となるウィンドウ切り替えコマンドです。

キーバインド アクション内容
Ctrl + E 次のウィンドウへ切り替え（Editor ➔ File Browser ➔ Menu を巡回）
Ctrl + Q 前のウィンドウへ切り替え
Ctrl + G ヘルプを開く（メニューのショートカット一覧へ即座にアクセス）

エディタ操作

nanoの操作性をベースとし、強力な検索・置換・カットバッファ機能を備えています。

キーバインド アクション内容
文字入力 カーソル位置に文字を挿入 (OS側のIMEを介して日本語入力にも対応)
Enter 改行を挿入
Backspace / Delete カーソルの前後の文字を削除 (行をまたいだ連結も可能)
↑ ↓ ← →
Home / End
PageUp / PageDown 		カーソルの移動
Ctrl + S ファイルを保存 (上書き保存)
Ctrl + O 名前を付けて保存 (Write Out; ファイル名の入力を求められます)
Ctrl + R カーソル位置に指定したファイルの内容を挿入 (Read File)
Ctrl + W ファイル内検索 (Where Is)。空欄でEnterを押すと前回の検索をリピート
Ctrl + \ 一括置換。検索文字列に続き、置換後文字列を入力して全一致箇所を置換
Ctrl + _ 指定した行番号へジャンプ (Go To Line)
Ctrl + K 現在の行をカット。連続してカットすると単一バッファに累積蓄積
Ctrl + U カットしたテキストを現在のカーソル位置に貼り付け (Uncut)
Ctrl + C 現在のカーソル位置情報 (行/列) および総行数をステータスに表示
Ctrl + X 終了。未保存の変更がある場合は、もう一度押して確認後に終了
Esc 検索、置換、ファイル名入力などの各種プロンプト入力をキャンセル

ファイルブラウザ

ファイルブラウザからファイルを確定すると、デフォルトで自動的にエディタへ切り替わります（設定で変更可能）。

キーバインド アクション内容
↑ / ↓ アイテムを選択
Enter / → 選択したディレクトリに入る、または選択したファイルを開く
Backspace / ← / u 親ディレクトリへ移動
g 絶対パス/相対パスを直接入力してジャンプ (カレントディレクトリ外も可)
n 新規ファイルの作成
N (Shift+n) 新規ディレクトリの作成
r 選択しているアイテムの名前変更
d 選択しているアイテムの削除 (y/nの確認プロンプトにより誤動作を防止)
Ctrl + H ドットファイル(隠しファイル)の表示/非表示切り替え
Esc 各種パス入力や名前変更プロンプトのキャンセル

メニューと設定ファイル

Wordライクな2ペイン構成のUIを採用。直感的な設定変更とTOMLファイルの直接編集が可能です。

キー操作
  • ← / →: 左側のカテゴリペインと右側の詳細ペインを切り替え
  • ↑ / ↓: 選択中ペイン内のアイテムを上下に移動
  • Enter: 左ペインから右ペインに遷移、または右ペインでオプションを選択/トグル

⚙️ 設定ファイルへのパス

メニューの「Configuration File」カテゴリから、TOMLファイル本体をエディタで開き直接変更することができます。設定は以下に即時反映・保存されます。 

~/.config/cirpath/config.toml

エラーメッセージの設計

Error: File 'invalid_path.txt' does not exist!

存在しないパスへの移動、保存失敗、検索語未検出などのエラーは、上記のように赤背景のステータスバーに表示されます。

💡 スマートな自動クリア設計:
エラーメッセージは、次に何らかのキーが入力された瞬間に自動消去されます。これにより古いエラー表示がいつまでも画面に残るノイズを防ぎ、快適なタイピング環境を維持します。
